# GRC-Alignment-Framework-
A technical framework designed to bridge the gap between high-level governance and operational security. This project simulates an enterprise security posture by mapping ISO 27001, NIST, GDPR, and SOC 2 requirements to granular technical controls.

# Project Overview
This repository serves as a comprehensive Governance, Risk, and Compliance (GRC) portfolio, demonstrating end-to-end enterprise risk management. It simulates the establishment of a baseline security and compliance posture for a fictional healthcare technology organization, mapping internal and vendor controls against industry-standard frameworks including ISO 27001, SOC 2 Type II, HIPAA, and NIST.

# Core Objectives
**Risk Management: **Identify, assess, and document enterprise IT risks, assigning inherent and residual risk scores alongside actionable mitigation strategies.

**Third-Party Risk Management (TPRM):** Design standardized Vendor Security Questionnaires (VSQs) to evaluate supply chain security postures.

**Compliance Auditing:** Perform simulated gap analyses and IT General Control (ITGC) validations to ensure regulatory alignment.

**Governance Documentation:** Draft core Information Security policies, standards, and access control procedures.

# Repository Structure
**01_Risk_Register/:** Contains the master IT & InfoSec Risk Register detailing 30 distinct enterprise threats and mitigations.

**02_Third_Party_Risk_Management/:** Vendor assessment workflows, VSQ templates, and vendor scoring matrices.

**03_Compliance_and_Audits/:** ISO 27001 Annex A Gap Analysis and control validation evidence.

**04_Policies_and_Procedures/:** Formal governance documents, including the Enterprise Access Control Policy.
